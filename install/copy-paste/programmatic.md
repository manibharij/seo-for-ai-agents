# Programmatic SEO — copy-paste mini (paste this into your AI coding agent)

*Self-contained version of the programmatic-SEO skill. Build pages at scale from data — the white-hat way.*

> ⚠️ **Experimental, and run by an AI agent — which can make mistakes.** This refuses thin/doorway mass-generation and never fabricates substance. Review the output on the served HTML. See the repo's `DISCLAIMER.md`.

---

You are helping me build pages at scale from a dataset — **the right way**. The line: **every generated page must offer genuine, distinct value, or it shouldn't exist.** If pages would differ only by a swapped variable with no real per-page substance, that's a doorway-page pattern — tell me, and recommend fewer/better pages instead of generating spam. Never fabricate substance to pad pages. Work in four steps.

## Step 1 — Assess the dataset & intent
Does the data give each page **genuinely distinct, useful content** (real differences), or near-identical templates? Is there **real search demand** for these page types (size it if a keyword tool is connected; otherwise reason and say so)? What's the **honest page count** — only entities with enough real data qualify? **If the dataset can't support distinct, useful pages, stop and tell me** — recommend fewer pages, enriching the data, or not doing it.

## Step 2 — Design
Template for consistent structure, but each page filled with **page-specific real content** (not boilerplate + swapped name). Lead with the searcher's value/answer. Plan honest schema per page (real values only). Plan hub/category pages + internal links + sitemap so the set is reachable, not orphaned. Plan **indexation/crawl-budget**: what's indexable vs noindex/canonical so you don't flood the index with low-value combos.

## Step 3 — Generate behind a quality gate
Generate from real data with a **quality threshold** — a page publishes only if it has enough real, distinct, useful content. **Pages below the bar are withheld or consolidated, never padded with fabricated text.** Keep it in version control; on a live site, roll out in batches and watch.

## Step 4 — Verify (served output)
Sample pages across the set: confirm real, distinct, substantive content (not thin/near-duplicate), honest schema, intent match, reachable/linked, in the sitemap. **Duplication check:** read several side by side — if near-identical, raise the bar or cut pages. **Crawl-budget check:** only the valuable subset is indexable. Report how many passed vs were withheld.

**The lines (non-negotiable):** no doorway pages, no fabricated substance, no mass-generation for the index's sake. "Generate 200 genuinely useful pages, not 20,000 thin ones" — or "don't" — is a valid, often correct answer.
